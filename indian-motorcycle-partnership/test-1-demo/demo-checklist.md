## ‚ö†Ô∏è TROUBLESHOOTING PROTOCOLS

### Issue: WiFi Connection Fails
**Resolution**:
- [ ] Switch to mobile hotspot immediately
- [ ] Announce backup plan to attendees calmly
- [ ] Verify API connectivity restored
- [ ] Continue demo without acknowledging as major issue

### Issue: App Crashes or Freezes
**Resolution**:
- [ ] Use backup device with app pre-loaded
- [ ] Restart app while explaining beta status
- [ ] Use demo videos as interim content
- [ ] Pivot to printed materials if needed

### Issue: AI Agent(s) Not Responding
**Resolution**:
- [ ] Check API key validity and quotas
- [ ] Verify internet connectivity
- [ ] Use cached example responses
- [ ] Show screenshots of working system
- [ ] Explain issue as temporary API limitation

### Issue: AR Overlay Not Working
**Resolution**:
- [ ] Show pre-recorded video of AR in action
- [ ] Display screenshots of AR overlay
- [ ] Explain technology with diagrams
- [ ] Focus on AI collaboration features instead
- [ ] Promise working AR demo in Test Phase 2

### Issue: Difficult Questions
**Resolution**:
- [ ] Listen carefully and don't interrupt
- [ ] Acknowledge validity of concern
- [ ] Be honest about limitations
- [ ] Commit to follow-up research if needed
- [ ] Turn challenge into opportunity for collaboration

### Issue: Skeptical Attendees
**Resolution**:
- [ ] Validate their expertise and experience
- [ ] Position as augmentation, not replacement
- [ ] Show specific value for their workflow
- [ ] Invite them to hands-on testing
- [ ] Focus on partnership vs. vendor relationship
